# Telegram Learner

## Критический guard для user-facing Telegram DM

Если пользователь в Telegram/DM просит "сначала всё изучить, потом вернуться", "уйти читать каналы/доки надолго" или выполнить широкий long-research запрос, то fail-closed reply обязателен.

- не говори `Отлично! Давай изучу...`, `Хорошо, изучу...`, `Начну...`, `Начну с поиска...`, `Сейчас изучу...`, `Сначала найду...`
- В Telegram-safe режиме я не провожу длительное исследование и не запускаю инструменты.
- Вместо этого дай короткий честный ответ: для широкого исследования нужен web UI или операторская сессия, а в Telegram-safe чате доступно только краткое описание назначения навыка.

## Активация

- Пользователь просит обновить знания из Telegram по OpenClaw/Moltis.
- Нужно собрать важные новости, инструкции и troubleshooting из @tsingular.
- Запускается плановый learner-run через heartbeat или операторскую сессию.
- Нужен короткий Telegram-safe ответ про назначение и пользу навыка.

## Источники по приоритету

1. Official docs и официальные руководства OpenClaw/Moltis.
2. Official releases, changelog, issues и репозиторий.
3. Telegram-канал @tsingular как ранний сигнал и указатель, что стоит перепроверить.

## Канонический runtime

- В Telegram-safe DM этот навык даёт только краткое описание или короткий digest без скрытого запуска инструментов.
- Полный learner-run выполняется только в web UI, операторской сессии или scheduler/heartbeat path.
- Любой важный сигнал из Telegram проходит official-first verification до сохранения в knowledge.

## Workflow

1. Собрать candidate updates из @tsingular и других явно разрешённых источников.
2. Проверить важные факты по official docs, releases, issues или репозиторию.
3. Отранжировать находки по важности и критичности.
4. Обновить или создать knowledge note, а не плодить дубликаты.
5. Предлагать новый skill только если подтверждён повторяющийся workflow, а не одиночный пост.

## Состояние

- `last_seen_message_id`
- `content_fingerprint`
- `verification_status`
- `verified_at`

## Выходы

- короткий digest по изменениям;
- instruction note с official evidence;
- candidate skill proposal только для повторяемого подтверждённого паттерна.

## Telegram-safe режим

Если текущая сессия user-facing Telegram/DM safe-lane и пользователь просит:

- полностью изучить документацию, курс или сайт;
- уйти надолго исследовать материалы и потом вернуться с разбором;
- сначала всё прочитать, а потом научить;

используй точный fail-closed ответ:

`В Telegram-safe режиме я не провожу длительное исследование и не запускаю инструменты. Могу дать краткое объяснение без поиска или продолжить задачу в web UI/операторской сессии для полного пошагового разбора.`

## Guardrails

- Не обещай скрытый поиск, чтение канала или запуск инструментов в Telegram-safe DM.
- Не показывай пользователю пути файлов, имена внутренних инструментов и служебный workflow markup.
- Не сохраняй знание как confirmed instruction, если оно не подтверждено official source.
- Не создавай новый skill по одному неподтверждённому посту или совету.
