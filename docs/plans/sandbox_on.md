## File/Config 1: `.claude/settings.json`

**Purpose:** Включение песочницы для Bash-операций и декларативное описание автономных прав (Allowlist) со строгими запретами на чувствительные данные (Zero Trust).
**Pattern:** Иерархическая конфигурация на уровне репозитория (Project Level). Bash-правила используют *префиксное совпадение* (не регулярные выражения).[^1]
**Reference:** See Permission Settings docs: code.claude.com/docs/settings

Создайте директорию `.claude/` в корне вашего проекта и добавьте туда файл `settings.json`:

```json
{
  "sandbox": {
    "enabled": true,
    "autoAllowBashIfSandboxed": true,
    "excludedCommands": ["docker", "git"]
  },
  "permissions": {
    "allow": [
      "Bash(curl *)",
      "Bash(wget *)",
      "Bash(npm run lint)",
      "Bash(npm run test)",
      "WebFetch",
      "Read"
    ],
    "ask": [
      "Bash(git push *)",
      "Bash(docker build *)"
    ],
    "deny": [
      "Read(./.env*)",
      "Read(./secrets/**)",
      "Read(./config/credentials.json)",
      "Bash(rm -rf *)",
      "Edit(./.github/workflows/**)"
    ]
  }
}
```

**Интеграция:**

- `autoAllowBashIfSandboxed` — автоматически подтверждает любые bash-команды (например `ls`, `grep`, `cat`), так как в macOS они исполняются в изолированном окружении.[^1]
- `allow: ["Bash(curl *)"]` — дает автономию на выполнение запросов к внешним API или скачивание ресурсов (песочница блокирует сеть, но это правило или встроенный `WebFetch` позволяют обойти ограничение для конкретных нужд).
- `ask` — принудительно запрашивает у вас `Y/n` (даже в обход песочницы) на отправку кода на сервер или сборку образов.
- `deny` — категорически запрещает агенту даже пытаться читать ваши секреты или изменять CI/CD-пайплайны (защита от "отравления" сборок).[^1]

## File/Config 2: `CLAUDE.md`

**Purpose:** Обеспечить контекстный контроль над автономным агентом. Это внутренний системный промпт проекта, который загружается при запуске `claude` в iTerm2.
**Pattern:** Явные декларативные инструкции (Prompt Engineering для управления поведением инструментов).
**Reference:** See Key points about the configuration system docs: code.claude.com/docs/settings

Создайте файл `CLAUDE.md` в корне вашего проекта:

```markdown
# Ограничения и правила автономности (Autonomy Policy)

1. **Безопасность данных (Zero Trust):** НИКОГДА не отправляй содержимое локальных файлов или фрагменты кода через `curl` или `WebFetch` во внешние API или сервисы, кроме явно запрошенных пользователем (например, для проверки синтаксиса на доверенном линтере).
2. **Использование сети:** При использовании `curl` или `WebFetch` для получения документации или сниппетов, всегда проверяй HTTP-статус и обрабатывай ошибки (таймауты, 404), чтобы не прерывать цепочку рассуждений.
3. **Файловая система:** Если нужно удалить файлы, используй безопасные скрипты или запрашивай подтверждение. Команды вроде `rm -rf` заблокированы политикой проекта.
```

## File/Config 3: Интеграция с GitOps (`.gitignore`)

**Purpose:** Предотвращение утечки локальных переопределений и защита инфраструктуры конфигурации.
**Pattern:** Стандартные практики контроля версий (Git).
**Reference:** See Settings files docs: code.claude.com/docs/settings

Добавьте следующие строки в ваш `.gitignore`:

```text
# Claude Code Local Overrides (Not Shared)
.claude/settings.local.json
.mcp.json
```

*(Примечание: `.claude/settings.json` и `CLAUDE.md` **нужно** закоммитить, чтобы правила применялись ко всем разработчикам проекта).*

### Verification (Проверка работоспособности)

1. Откройте ваш проект в **iTerm2**.
2. Запустите `claude`.
3. Введите команду: `/config`
    - Убедитесь, что в интерфейсе настройки песочницы и разрешений подтянулись из "Project Settings" (или "Shared").
4. Введите тестовый промпт:
*«Скачай главную страницу example.com через curl, прочитай файл package.json и выведи их размеры. Также попытайся прочитать файл .env».*
5. **Ожидаемый результат:**
    - `curl` выполнится автономно без запроса Y/n.
    - `package.json` прочитается автономно.
    - На попытку прочесть `.env` Claude Code выдаст ошибку `Permission Denied` из-за правил блокировки.

### Troubleshooting (Решение проблем)

- **Проблема:** Команды `curl` или `npm` не работают (виснет или падает с ошибкой Command not found).
- **Фикс:** В macOS песочница может изолировать пути к исполняемым файлам (например, установленным через Homebrew в `/opt/homebrew/bin`). В таком случае вам может потребоваться передать окружение через `CLAUDE_ENV_FILE` (создав скрипт, который экспортирует `$PATH` перед запуском команд), либо временно добавить команду в `excludedCommands` песочницы. See Bash tool behavior docs: code.claude.com/docs/tools.[^1]
