# RCA: Token Bloat в инструкциях — повторяющаяся проблема

**Дата:** 2026-03-04
**Статус:** Resolved
**Влияние:** Потеря контекста, увеличенные расходы на токены, замедление работы

## Ошибка

Центральные файлы инструкций (CLAUDE.md ~700 строк, MEMORY.md ~300 строк) продолжают расти, хотя проблема token bloat уже обсуждалась и "решалась" через:
- Lessons Architecture (distributed lessons, centralized index)
- Минимальные ссылки в CLAUDE.md на LESSONS-LEARNED.md

## Анализ 5 Почему

| Уровень | Вопрос | Ответ |
|---------|--------|-------|
| 1 | Почему файлы разрастаются? | Новый контент добавляется напрямую в CLAUDE.md/MEMORY.md |
| 2 | Почему добавляется напрямую? | Нет явного запрета "НЕ добавлять — создавать отдельные файлы" |
| 3 | Почему нет запрета? | Правило есть в LESSONS-LEARNED.md, но не в **самих файлах** как header |
| 4 | Почему не в самих файлах? | Добавили reference, но не **explicit prohibition** с max size |
| 5 | Почему это повторяется? | LLM не помнит предыдущие сессии → нет persistent правила в файле |

## Корневая причина

**Правила token optimization находятся в ДРУГИХ файлах**, а не в тех, которые нужно ограничить.

LLM в новой сессии:
1. Читает CLAUDE.md → не видит ограничения размера
2. Добавляет новый контент напрямую
3. Файл растёт → токены тратятся

## Принятые меры

### 1. Немедленное исправление

Добавлен **HTML comment header** в оба файла:

```html
<!--
⛔ TOKEN BLOAT PREVENTION (RCA-004)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
MAX SIZE: 700 lines | Current: ~690 lines

⚠️ НЕ ДОБАВЛЯТЬ новый контент напрямую!
→ Создай отдельный файл в docs/ или .claude/
→ Добавь только КРАТКУЮ ссылку (1-2 строки)

WHY? Каждый лишний токен потребляется при КАЖДОМ запросе.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
-->
```

### 2. Предотвращение

| Файл | Max Size | Куда добавлять новый контент |
|------|----------|------------------------------|
| CLAUDE.md | 700 lines | docs/, .claude/rules/, .claude/skills/ |
| MEMORY.md | 300 lines | docs/LESSONS-LEARNED.md, docs/rca/ |

### 3. Документация

- Это RCA фиксирует проблему как **recurring issue**
- Header в файлах работает как **persistent reminder** для каждой новой сессии

## Уроки

1. **Правила должны быть В САМИХ файлах**, которые они ограничивают — не в "связанных" файлах
2. **LLM не имеет памяти между сессиями** — persistent rules должны быть в контенте
3. **HTML comments** — хороший способ добавить инструкции для LLM, не влияя на рендеринг
4. **Explicit prohibition > implicit reference** — "НЕ добавлять" лучше чем "см. LESSONS-LEARNED.md"
5. **Max size в строках** — конкретное ограничение лучше абстрактного "не разрастайся"
