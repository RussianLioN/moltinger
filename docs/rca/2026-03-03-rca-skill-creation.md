# RCA: Тестирование навыка rca-5-whys

**Дата:** 2026-03-03
**Статус:** Resolved
**Влияние:** Нет влияния на продакшен (тестовый RCA для демонстрации)
**Контекст:** Создание и тестирование нового навыка

## Ошибка

При создании навыка возникла ошибка `Operation not permitted` при попытке создать директорию через `mkdir` в sandbox-режиме.

## Анализ 5 Почему

| Уровень | Вопрос | Ответ |
|---------|--------|-------|
| 1 | Почему mkdir вернул "Operation not permitted"? | Команда выполнялась в sandbox-режиме, который блокирует операции с файловой системой вне разрешённых директорий |
| 2 | Почему sandbox блокирует создание директорий? | Sandbox изолирует Bash команды для безопасности, требуя явного подтверждения или флага `dangerouslyDisableSandbox` |
| 3 | Почему не был использован флаг dangerouslyDisableSandbox? | Не было явной проверки sandbox-статуса перед выполнением команды |
| 4 | Почему нет автоматической обработки sandbox-ограничений? | В инструкциях есть раздел про sandbox, но нет явного паттерна "попробуй sandboxed → если ошибка → retry с dangerouslyDisableSandbox" |
| 5 | Почему нет такого паттерна? | Паттерн не был зафиксирован как явная инструкция в "Core Rules" |

## Корневая причина

Отсутствие явного паттерна обработки sandbox-ошибок в инструкциях выполнения команд. Sandbox работает корректно, но процесс восстановления из sandbox-ошибки не формализован.

## Принятые меры

1. **Немедленное исправление:** Использован `dangerouslyDisableSandbox: true` для создания директории
2. **Предотвращение:** Документирован паттерн в этом RCA
3. **Документация:** Создан навык rca-5-whys для будущих анализов
