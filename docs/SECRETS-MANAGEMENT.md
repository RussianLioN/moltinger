# Secrets Management Policy

**Version**: 1.0
**Last Updated**: 2026-03-07

---

## 🎯 Principles

1. **GitOps First**: Все изменения через git + CI/CD
2. **No Hardcoded Secrets**: Никаких секретов в коде
3. **Single Source of Truth**: GitHub Secrets как центральное хранилище
4. **Environment Parity**: Одинаковые секреты для всех окружений

## ✅ Allowed Patterns

### 0. Context-first lookup (for agents and operators)

Before asking for secret/env values, check:

1. `MEMORY.md`
2. `SESSION_SUMMARY.md`
3. This file (`docs/SECRETS-MANAGEMENT.md`)
4. `.github/workflows/deploy.yml` (`Generate .env from Secrets`)

If a value is already documented there, reuse it and do not ask again.

### 1. GitHub Secrets (Recommended)

```bash
# Set secret
gh secret set SECRET_NAME --repo owner/repo

# Use in GitHub Actions
env:
  GLM_API_KEY: ${{ secrets.GLM_API_KEY }}
```

### 2. Environment Variables via CI/CD

```yaml
# .github/workflows/deploy.yml
- name: Create .env file
  run: |
    echo "GLM_API_KEY=${{ secrets.GLM_API_KEY }}" > .env
    echo "BRAVE_API_KEY=${{ secrets.BRAVE_API_KEY }}" >> .env
    scp .env $SSH_USER@$SSH_HOST:$DEPLOY_PATH/.env
```

### 3. Local Development (.env.local)

```bash
# .env.local (git-ignored)
cp .env.example .env.local
# Edit with real values
```

## ❌ Forbidden Patterns

### 1. Hardcoded Secrets in Code

```toml
# ❌ NEVER DO THIS
api_key = "71c37e69c6f7503fb6480d7b..."

# ✅ DO THIS INSTEAD
api_key = "${ELEVENLABS_API_KEY}"
```

### 2. Manual scp of Secrets

```bash
# ❌ NEVER DO THIS
scp .env root@server:/opt/app/.env

# ✅ DO THIS INSTEAD (from CI/CD)
# In GitHub Actions workflow:
- name: Sync secrets
  run: scp .env $SSH_USER@$SSH_HOST:$DEPLOY_PATH/.env
```

### 3. Editing Secrets on Server

```bash
# ❌ NEVER DO THIS
ssh root@server "vim /opt/app/.env"

# ✅ DO THIS INSTEAD
# 1. Update GitHub Secret
gh secret set NEW_KEY --repo owner/repo
# 2. Trigger redeploy
gh workflow run deploy.yml
```

### AI Workflow Secrets (Code Review + Interactive Assistant)

- `.github/workflows/claude-code-review.yml` and `.github/workflows/claude.yml` use `GLM_API_KEY` for Z.ai Coding Plan requests.
- `CLAUDE_CODE_OAUTH_TOKEN` and `ANTHROPIC_API_KEY` are not required for current CI workflows.
- Optional kill-switch: repository variable `AI_REVIEW_PROVIDER=off` disables active AI calls while keeping fallback reports/comments non-blocking.
- Rollback artifacts for legacy Anthropic workflows are stored in:
  - `.github/workflows/claude.legacy.yml.disabled`
  - `.github/workflows/claude-code-review.legacy.yml.disabled`

For on-demand Telegram E2E harness:
- `synthetic` mode requires only `MOLTIS_PASSWORD`.
- `real_user` mode requires `TELEGRAM_TEST_API_ID`, `TELEGRAM_TEST_API_HASH`, and `TELEGRAM_TEST_SESSION`; `TELEGRAM_TEST_BOT_USERNAME` is optional.
- `TELEGRAM_TEST_SESSION` is generated once via `scripts/telegram-real-user-bootstrap.py` (OTP login).

### LLM Failover Secrets

The Ollama fallback system requires the following secret:

| Secret | Required | Purpose |

**When OLLAMA_API_KEY is needed:**
- Using Ollama Cloud models with `:cloud` suffix
- Not required for local Ollama models

**To add OLLAMA_API_KEY:**
```bash
gh secret set OLLAMA_API_KEY --repo RussianLioN/moltinger
```

## 🔄 Workflow: Adding New Secret

1. **Generate secret value** (API key, password, etc.)
2. **Add to GitHub Secrets:**
   ```bash
   gh secret set NEW_SECRET --repo owner/repo
   ```
3. **Update .env.example** (with placeholder):
   ```bash
   echo "# NEW_SECRET=your-value-here" >> .env.example
   ```
4. **Update moltis.toml** (if needed):
   ```toml
   api_key = "${NEW_SECRET}"
   ```
5. **Update CI/CD workflow** (if needed):
   ```yaml
   env:
     NEW_SECRET: ${{ secrets.NEW_SECRET }}
   ```
6. **Commit and push** - CI/CD deploys automatically

## 🚨 Incident Response

### If Secret is Committed to Git

1. **Immediately rotate the secret** (generate new API key)
2. **Update GitHub Secret** with new value
3. **Force push to remove from history** (if critical)
4. **Review access logs** for unauthorized usage

### Commands for Rotation

```bash
# 1. Generate new key at provider's dashboard

# 2. Update GitHub Secret
gh secret set COMPROMISED_KEY --repo owner/repo

# 3. Trigger redeploy
gh workflow run deploy.yml

# 4. Verify
gh run watch
```
