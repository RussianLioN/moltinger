# Глоссарий: ASC AI Fabrique

**Версия**: v0.1 (Черновик)
**Дата**: 2025-11-05

## Обзор

Данный глоссарий содержит ключевые термины и концепции, используемые в документации ASC AI Fabrique. Термины организованы в алфавитном порядке с указанием английского перевода (где применимо) и связей с другими документами.

**Примечание**: Перекрестные ссылки на конкретные разделы документов будут добавлены на этапе 3.

## А

### Агент-архитектор
**Англ.**: Agent-Architect  
**Определение**: Роль ИИ-агента, ответственного за проектирование архитектуры новых ИИ-агентов с учетом требований безопасности, производительности и соответствия нормам.  
**Используется в**: self_referential_mapping.md, strategic_roadmap.md  
**Связанные термины**: [Метаблок](#метаблок), [ARCHITECTURE_PATTERN](#architecture_pattern)

### Агент-генератор
**Англ.**: Agent-Generator  
**Определение**: Роль ИИ-агента, ответственного за автоматическое создание новых ИИ-агентов с применением метаблоков и паттернов.  
**Используется в**: self_referential_mapping.md  
**Связанные термины**: [Фабрика ИИ-агентов](#фабрика-ии-агентов), [AGENT_DEVELOPMENT_PATTERN](#agent_development_pattern)

### Агент-документатор
**Англ.**: Agent-Documentor  
**Определение**: Роль ИИ-агента, ответственного за автоматическое создание и поддержание документации для новых ИИ-агентов.  
**Используется в**: self_referential_mapping.md  
**Связанные термины**: [DEVELOPMENT_PATTERN](#development_pattern)

**Используется в**: self_referential_mapping.md, strategic_roadmap.md

**Англ.**: Agent-Researcher

## Г

### GPU (Графический процессор)
**Англ.**: GPU (Graphics Processing Unit)
**Определение**: Специализированный процессор для параллельной обработки данных, используемый для инференса больших языковых моделей. В ASC AI Fabrique применяются серверы с 8× NVIDIA A100/H100.
**Используется в**: devops_infrastructure_tz.md, Запрос железа...
**Связанные термины**: [NVLink](#nvlink), [Tensor Parallelism](#tensor-parallelism), [Inference](#inference-инференс)

### GLM-4.7
**Англ.**: GLM-4.7
**Определение**: Большая языковая модель от Zhipu AI (Z.AI) с 355B параметров (MoE архитектура, 32B активных), поддерживающая 200,000 токенов контекста. Используется в ASC AI Fabrique в фазе 0.
**Используется в**: devops_infrastructure_tz.md
**Связанные термины**: [MoE](#moe-mixture-of-experts), [Inference](#inference-инференс), [vLLM](#vllm)

## К

### Квантование (Quantization)
**Англ.**: Quantization
**Определение**: Метод снижения точности весов нейросети для уменьшения требований к памяти и вычислительным ресурсам. Форматы: FP8 (<1% потерь), BF16 (полная точность), AWQ 4bit (7-10% потерь).
**Используется в**: devops_infrastructure_tz.md
**Связанные термины**: [GPU](#gpu-графический-процессор), [Inference](#inference-инференс), [vLLM](#vllm)

## М

### Матрица решений
**Англ.**: Decision Matrix
**Определение**: Таблица для сравнительной оценки вариантов решений по набору критериев с весовыми коэффициентами.

### MCP (Model Context Protocol)
**Англ.**: Model Context Protocol
**Определение**: Стандарт для структурированного доступа ИИ-моделей к внешним инструментам и системам. GLM-4.7 поддерживает MCP нативно, что упрощает интеграцию с корпоративными системами.
**Используется в**: devops_infrastructure_tz.md
**Связанные термины**: [GLM-4.7](#glm-47), [Tool Calling](#tool-calling)

### Метаблок
**Англ.**: Meta-Block
**Определение**: Повторно используемый шаблон или паттерн, применяемый при создании ИИ-агентов. Описывает структурированный подход к решению определенного типа задач.
**Используется в**: Все документы
**Связанные термины**: [Реестр метаблоков](#реестр-метаблоков), [Семенные паттерны](#семенные-паттерны)

### MoE (Mixture of Experts)
**Англ.**: Mixture of Experts
**Определение**: Архитектура нейросети, где только часть параметров активна на каждом инференсе. GLM-4.7 имеет 355B параметров, но активны только 32B, что снижает требования к памяти.
**Используется в**: devops_infrastructure_tz.md
**Связанные термины**: [GLM-4.7](#glm-47), [Квантование](#квантование-quantization), [Inference](#inference-инференс)

### Мониторинг
**Англ.**: Monitoring
**Определение**: Система сбора и анализа метрик о работе инфраструктуры и приложений. В ASC AI Fabrique используется Prometheus + Grafana + DCGM Exporter для мониторинга GPU.
**Используется в**: devops_infrastructure_tz.md
**Связанные термины**: [DCGM](#dcgm-data-center-gpu-manager), [GPU](#gpu-графический-процессор)

### MVP0
**Англ.**: Minimum Viable Product 0
**Определение**: Минимально жизнеспособный продукт, создаваемый в фазе 0 (<4 недели) для демонстрации концепции фабрики.
**Используется в**: strategic_roadmap.md, defense_poc_template.md
**Связанные термины**: [Фаза 0](#фаза-0), [Defense Gate](#defense-gate)

**Англ.**: FP8
**Определение**: Формат квантования с 8-битной точностью с плавающей запятой. Обеспечивает минимальную потерю точности (<1%) при максимальной производительности. Рекомендуется для GLM-4.7 на H100.

### BF16 (Brain Float 16)
**Англ.**: BF16
**Определение**: Формат данных с 16-битной точностью с плавающей запятой (Bfloat16). Обеспечивает полную точность вычислений при хорошем балансе производительности. Требует больше памяти, чем FP8 или AWQ 4bit.
**Используется в**: devops_infrastructure_tz.md
**Связанные термины**: [Квантование](#квантование-quantization), [FP8](#fp8-float-point-8-bit), [GPU](#gpu-графический-процессор)
