# Реестр метаблоков: ASC AI Fabrique

**Версия**: v1.0
**Дата**: 2025-10-30
**Статус**: Черновик

***

## Обзор

Данный документ описывает схему и основу реестра метаблоков для ASC AI Fabrique - системы автоматического самовоспроизводящегося и самосовершенствующегося разработки. Реестр метаблоков представляет собой репозиторий повторно используемых шаблонов и паттернов, которые используются фабрикой при создании ИИ-агентов.

В этом документе определена схема реестра, представлены начальные семенные паттерны, а также описаны процессы извлечения и применения метаблоков.

---

## Схема реестра метаблоков

### Структура метаблока

Каждый метаблок в реестре представлен в следующей структуре:

```json
{
  "id": "Уникальный идентификатор метаблока (например, RESEARCH_FRAMEWORK_PATTERN)",
  "name": "Человекочитаемое имя метаблока",
  "category": "Категория (например, RESEARCH, ARCHITECTURE, DEVELOPMENT, TESTING, etc.)",
  "subcategory": "Подкатегория (например, FRAMEWORK_EVALUATION, ARCHITECTURE_DESIGN)",
  "description": "Описание метаблока и его назначения",
  "context": "Контекст, в котором метаблок применяется",
  "workflow": {
    "phases": [
      {
        "phase_name": "Название фазы",
        "tasks": ["Список задач"],
        "deliverables": ["Список результатов"],
        "success_criteria": ["Критерии успеха"]
      }
    ]
  },
  "implementation_notes": "Рекомендации по реализации",
  "ai_acceleration": {
    "tools": ["Список ИИ-инструментов, применимых к метаблоку"],
    "expected_gain": "Ожидаемое ускорение (например, 2-3x)"
  },
  "dependencies": ["Список зависимых метаблоков"],
  "metrics": {
    "performance_indicators": ["Показатели эффективности"],
    "quality_indicators": ["Показатели качества"]
  },
  "version": "Версия метаблока",
  "author": "Автор метаблока",
  "created_date": "Дата создания",
  "last_updated": "Дата последнего обновления",
```

## Руководство по извлечению метаблоков

### Процесс извлечения

Метаблоки могут быть извлечены из успешных проектов, компонентов и процессов по следующему процессу:

1. **Идентификация успешных паттернов**
   - Анализ успешных проектов и компонентов
   - Определение повторяющихся подходов и решений
   - Оценка применимости в других контекстах

2. **Документирование паттерна**
   - Использование стандартной схемы метаблока
   - Подробное описание контекста и применимости
   - Определение зависимостей и интеграций

3. **Валидация и рецензирование**
   - Проверка точности и полноты документации
   - Рецензирование другими специалистами
   - Тестирование применимости в другом контексте

4. **Регистрация в реестре**
   - Назначение уникального ID и версии
   - Добавление в центральный реестр
   - Обновление связей и зависимостей

### Автоматическое извлечение

В будущем, когда система станет более зрелой:

- ИИ-агенты могут автоматически идентифицировать успешные паттерны
- Система может предложить формализацию и регистрацию
- Автоматизация процесса извлечения и документирования
