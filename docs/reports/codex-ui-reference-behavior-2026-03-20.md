## Цель

Зафиксировать наблюдаемое UX-поведение референсного интерфейса как формальные пользовательские требования для `ASC AI Fabrique Demo`.

## Наблюдаемые сценарии (эталон)

1. Левая и правая боковые панели открываются и закрываются через кнопки в хедере.
2. На разделителях панелей есть интерактивное состояние при наведении (визуальная иконка/маркер ресайза).
3. Двойной клик по разделителю возвращает ширину боковой панели к дефолтной.
4. Разделитель поддерживает drag-resize:
   - левая панель регулируется в ограниченном диапазоне;
   - правая панель может раскрываться существенно шире;
   - при раскрытии правой панели центральная и противоположная области сжимаются до минимально допустимых размеров.
5. Правая панель имеет действие разворота в полный экран (maximized focus mode).

## UX Контракт (пользовательские требования)

### US-01: Управление боковыми панелями кнопками

Как пользователь, я хочу одной кнопкой скрывать/показывать боковые панели, чтобы освободить пространство под основной контент.

Acceptance Criteria:
- Given панель открыта, when пользователь нажимает кнопку панели, then панель скрывается.
- Given панель скрыта, when пользователь нажимает кнопку панели, then панель появляется.
- Then центральная область корректно перерассчитывает доступную ширину без перекрытий.

### US-02: Hover-состояние разделителя

Как пользователь, я хочу видеть явный признак интерактивности разделителя при наведении, чтобы понимать, что панель можно менять по ширине.

Acceptance Criteria:
- Given курсор вне разделителя, then разделитель отображается в нейтральном состоянии.
- Given курсор наведен на разделитель, then появляется визуальный маркер ресайза.
- Then курсор и/или маркер соответствует ожидаемому типу изменения размера.

### US-03: Double-click reset ширины

Как пользователь, я хочу двойным кликом сбрасывать ширину панели к дефолтной, чтобы быстро возвращаться к базовой компоновке.

Acceptance Criteria:
- Given ширина панели изменена, when пользователь делает двойной клик по разделителю панели, then ширина панели устанавливается в дефолтное значение.
- Then соседние области интерфейса перерассчитываются без скачков верстки.

### US-04: Drag-resize с ограничениями

Как пользователь, я хочу тянуть разделитель панелей, чтобы гибко настраивать рабочее пространство.

Acceptance Criteria:
- Given drag начат по разделителю, when курсор двигается, then ширина соответствующей панели изменяется в реальном времени.
- Given левая панель, then ширина ограничена min/max.
- Given правая панель, then допускается широкое раскрытие вплоть до максимально полезного диапазона.
- Then центральная и противоположная области не схлопываются ниже допустимого минимума.

### US-05: Fullscreen для правой панели

Как пользователь, я хочу развернуть правую панель на весь экран, чтобы работать с артефактом без визуального шума.

Acceptance Criteria:
- Given правая панель доступна, when пользователь нажимает кнопку fullscreen, then включается режим максимального фокуса на правой панели.
- When пользователь выходит из fullscreen, then восстанавливается предыдущая рабочая компоновка.

## Нефункциональные требования UX

1. Все действия открытия/закрытия/ресайза должны отрабатывать без визуальных артефактов и перекрытий.
2. Разделители должны иметь достаточно широкую hit-area для удобного захвата мышью.
3. Состояния панелей должны быть консистентны после reload (в рамках выбранной стратегии персистентности).
4. В mobile/narrow viewport поведение должно деградировать предсказуемо (без «сломанных» колонок).

## Трассируемость в тесты

Рекомендуемая матрица автопроверок:

- `PW-PANEL-TOGGLE-L`: toggle левой панели кнопкой.
- `PW-PANEL-TOGGLE-R`: toggle правой панели кнопкой.
- `PW-DIVIDER-HOVER-L`: hover разделителя левой панели.
- `PW-DIVIDER-HOVER-R`: hover разделителя правой панели.
- `PW-DIVIDER-DBLCLICK-L`: double-click reset левой панели.
- `PW-DIVIDER-DBLCLICK-R`: double-click reset правой панели.
- `PW-DIVIDER-DRAG-L`: drag-resize левой панели в min/max диапазоне.
- `PW-DIVIDER-DRAG-R`: drag-resize правой панели и проверка сжатия центральной области.
- `PW-RIGHT-PANEL-FULLSCREEN`: вход/выход из fullscreen правой панели.
