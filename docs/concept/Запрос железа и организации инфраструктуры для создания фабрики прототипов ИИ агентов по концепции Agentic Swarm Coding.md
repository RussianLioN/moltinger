# Запрос железа и организации инфраструктуры для создания фабрики прототипов ИИ агентов по концепции Agentic Swarm Coding

## ЗАПРОС  
  
Просим рассмотреть возможность предоставить GPU-инфраструктуру для двухэтапного проекта автоматизированной фабрики генерации прототипов ИИ-агентов.  
  
**Что требуется:**  
  
**Этап 1 (Q4 2025, 3 месяца) — Валидация гипотез:**  
  
- Единый сервер 8× NVIDIA A100 80GB SXM с NVLink 3.0 (минимум)  
- Или 8× NVIDIA H100 80GB SXM с NVLink 4.0 (желательно)  
- Bare-metal под наше управление  
  
**Этап 2 (Q1 2026, 3 месяца) — Тестирование роя:**  
  
- Расширение до 16× GPU (A100 или H100/H200/B200, в зависимости от Этапа 1)  
- Единая конфигурация: 2 сервера по 8 GPU или 1 сервер на 16 GPU  
  
**Обязательные требования (оба этапа):**  
  
- Мультипользовательский доступ для 20-40 одновременных пользователей  
- Организация разделенного хранилища для пользователей  
- Поддержка Kubernetes multi-tenancy и GPU time-slicing**Ожидаем от вас:**  
  
1. Подтверждение технической возможности предоставления для первого и второго этапов  
2. Сроки развёртывания для каждого этапа  
3. Возможность расширения 8 → 16 GPU  
4. Ответы на вопросы из раздела 4

## 1. ОБОСНОВАНИЕ ЗАДАЧИ И ПОЭТАПНЫЙ ПЛАН  
  
Разрабатываем автоматизированную фабрику генерации прототипов ИИ-агентов (Agentic Swarm Coding): рой из 4-6 специализированных ИИ-агентов автономно создаёт других ИИ агентов через цепочку: анализ требований → архитектура → код → тесты → аудит безопасности → развёртывание.  
  
**Результат:** Готовый прототип агента за 4-12 часов (против 3-6 месяцев традиционной разработки).  
  
### Поэтапная реализация:  
  
**Q4 2025 (8 GPU) — Валидация гипотез:**  
  
- Цель: Проверить работоспособность двух ключевых моделей (GLM-4.6 и Qwen3-Coder-480B)  
- Результат: Подтверждение возможности генерации кода через базовую цепочку агентов  
- Ограничение: Контекст 8-16K токенов (достаточно для proof-of-concept)  
  
**Q1 2026 (16 GPU) — Тестирование роя:**  
  
- Цель: Запуск полноценного роя из 4-6 агентов с контекстом 128K токенов  
- Результат: 3-5 работающих агентов в продакшн  
- Возможность: Параллельная работа нескольких роев

## 2. ТРЕБОВАНИЯ К КОНФИГУРАЦИИ  
  
### 2.1 Почему единый сервер (критично для каждого 8-GPU модуля)  
  
|Критерий|Единый 8-GPU сервер|Распределенный кластер|Почему критично|  
|---|---|---|---|  
|Латентность GPU-GPU|<2 мкс (NVLink)|10-50 мкс (сеть)|Tensor parallelism для моделей 380B-480B|  
|NVLink топология|All-to-all (полная связность)|Только внутри узла|Эффективная передача тензоров 190-480 ГБ|  
|Управление|Простое (1 узел)|Сложное (синхронизация)|Упрощение для proof-of-concept|  
  
**Для 16 GPU:** Предпочтительно 2 сервера по 8 GPU с высокоскоростным межсерверным соединением (InfiniBand HDR 200 Гбит/с).  
  
### 2.2 Спецификация сервера (на каждые 8 GPU)  
  
|Компонент|A100 (минимум)|H100/H200/B200 (желательно)|  
|---|---|---|  
|GPU|8× A100 80GB SXM|8× H100 80GB SXM (+217% производительность)|  
|NVLink|3.0 (600 ГБ/с)|4.0 (900 ГБ/с)|  
|CPU|2× AMD EPYC 9554 (128 ядер)|2× AMD EPYC 9654 (192 ядра)|  
|RAM|2 ТБ DDR5 ECC|2 ТБ DDR5 ECC|  
|NVMe|20 ТБ PCIe 4.0+|20 ТБ PCIe 5.0|  
|Сеть|2× 100 Гбит/с + изоляция|2× 100 Гбит/с + изоляция|  
  
**Для 16 GPU:** Межсерверная связь через InfiniBand HDR (200 Гбит/с) или RoCE v2 (100 Гбит/с).  
  
### 2.3 Мультипользовательский доступ  
  
**Сценарий:** 20-40 одновременных пользователей (инженеры, исследователи, аналитики).  
  
**Обязательные возможности:**  
  
- **Изоляция:** LDAP/OAuth, Kubernetes namespace, resource quotas  
- **Хранилище:** изолированные каталоги 500 ГБ - 2 ТБ на пользователя  
- **GPU:** time-slicing, приоритизация, мониторинг по пользователям  
- **Защита:** OOM одного пользователя не влияет на других  
  
**Требуется поддержка:**  
  
- JupyterHub или аналог  
- Kubernetes multi-tenancy  
- Система очередей (Slurm / K8s native)---  
  
## 3. МОДЕЛИ И ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ  
  
### 3.1 Ключевые модели и требования к памяти  
  
|Модель|Параметры|FP8 (H100)|AWQ 4bit (A100)|Роль|  
|---|---|---|---|---|  
|**GLM-4.6**|**~380B**|**380 ГБ** (~5 GPU)|**190 ГБ** (~3 GPU)|Анализ требований, контекст 128K|  
|**Qwen3-Coder-480B**|**~480B**|**480 ГБ** (~6 GPU)|**240 ГБ** (~3 GPU)|Генерация кода|  
  
### 3.2 Распределение ресурсов по этапам  
  
**Q4 2025 (8 GPU) — Валидация гипотез:**  
  
|Конфигурация|Модели|Память занята|Резерв|Контекст|  
|---|---|---|---|---|  
|**8× H100**|GLM-4.6 (380 ГБ) + Qwen3-480B (480 ГБ)|860 ГБ|❌ **Не влезает**|-|  
|**8× A100**|GLM-4.6 (190 ГБ) + Qwen3-480B (240 ГБ)|430 ГБ|210 ГБ|8-16K токенов|  
  
**Вывод Q4 2025:** На 8 GPU возможна только базовая валидация с ограниченным контекстом.  
  
**Q1 2026 (16 GPU) — Полноценный рой:**  
  
|Конфигурация|Модели + Рой|Память занята|Резерв|Контекст|  
|---|---|---|---|---|  
|**16× H100**|GLM-4.6 (380) + Qwen3-480B (480) + 3× Qwen3-30B (48) + 4× Qwen3-7B (20)|~928 ГБ|**352 ГБ**|**128K токенов** ✅|  
|**16× A100**|GLM-4.6 (190) + Qwen3-480B (240) + 3× Qwen3-30B (54) + 4× Qwen3-7B (20)|~504 ГБ|**776 ГБ**|**128K токенов** ✅|  
  
**Вывод Q1 2026:** 16 GPU обеспечивают полноценную работу роя с контекстом 128K токенов.  
  
### 3.3 Критичные функции моделей  
  
**Для автономной работы роя необходимы:**  
  
1. **Длинный контекст (128K токенов)** — анализ целых микросервисов без фрагментирования  
2. **Chain-of-thought reasoning** — пошаговое планирование (+40-60% качество)  
3. **Богатый tool calling** — взаимодействие с `git`, `pytest`, `docker`, БД для автономной работы  
4. **MCP (Model Context Protocol)** — стандартизованный доступ к корпоративным системам без хардкода credentials  
5. **A2A коммуникация (Agent-to-Agent)** — передача структурированных данных между агентами роя  
6. **Квантизация** — FP8 (H100, потеря 2-3%) или AWQ 4bit (A100, потеря 7-10%)  
7. **Мультиязычность** — русский (требования) + английский (код)  
8. **Структурированный вывод** — валидный JSON Schema
