## Конкретное ПО для разворачивания инфраструктуры (тип + 3 альтернативы)

### **Тип: Платформа контейнерной оркестрации**
- **Основной**: **Kubernetes** (управление GPU-нодами, auto-scaling, RBAC)
- **Альтернатива 1**: **OpenShift** (enterprise Kubernetes от Red Hat, лучшая безопасность)
- **Альтернатива 2**: **Rancher** (упрощенный UI, подходит для небольших команд)
- **Альтернатива 3**: **Nomad** (HashiCorp, проще Kubernetes, но меньше экосистема)

### **Тип: Система потоковой обработки событий**
- **Основной**: **Redpanda** (Kafka-совместимый, на C++, минимальная latency)
- **Альтернатива 1**: **Apache Kafka** (стандарт де-факто, большая экосистема)
- **Альтернатива 2**: **NATS JetStream** (легкий, встроенный в Kubernetes)
- **Альтернатива 3**: **Apache Pulsar** (много-арендная модель, хорош для роев)

### **Тип: Inference Engine для LLM**
- **Основной**: **vLLM** (PagedAttention, quantization, простота)
- **Альтернатива 1**: **TensorRT-LLM** (NVIDIA, максимальная скорость, сложная настройка)
- **Альтернатива 2**: **Text Generation Inference (TGI)** (HuggingFace, хорош для мультимодальных)
- **Альтернатива 3**: **Ray Serve** (распределенный инференс, если модель не влезает в один GPU)

### **Тип: Фреймворк оркестрации агентов**
- **Основной**: **LangGraph** (stateful графы, persistence, критично для фабрики)
- **Альтернатива 1**: **Semantic Kernel** (Microsoft, интеграция с 365, но менее гибкий)
- **Альтернатива 2**: **CrewAI** (rapid prototyping роев, но переходим на LangGraph в прод)
- **Альтернатива 3**: **AutoGen** (Microsoft, диалоговые рои, для этических агентов)

### **Тип: Векторная база данных**
- **Основной**: **Qdrant** (Rust, быстрый, фильтры, on-premise)
- **Альтернатива 1**: **pgvector** (Postgres, проще, если уже есть Postgres)
- **Альтернатива 2**: **ClickHouse + ANN** (если нужны аналитика и вектора вместе)
- **Альтернатива 3**: **Milvus** (Zilliz, распределенный, но тяжелый)

### **Тип: Система мониторинга**
- **Основной**: **Prometheus + Grafana** (стандарт, open-source, GPU‑метрики)
- **Альтернатива 1**: **VictoriaMetrics** (быстрее Prometheus, меньше ресурсов)
- **Альтернатива 2**: **Zabbix** (старое добро, хорошо знают корпоративные SRE)
- **Альтернатива 3**: **ELK Stack** (если нужен поиск по логам, но тяжелый)

### **Тип: Трассировка LLM (observability)**
- **Основной**: **LangSmith** (self-hosted, платная лицензия, но лучший для LangGraph)
- **Альтернатива 1**: **Helicone** (open-source, самостоятельный хостинг)
- **Альтернатива 2**: **DeepEval** (оценка качества, faithfulness, полностью open)
- **Альтернатива 3**: **Arize Phoenix** (трейсинг + датасеты для fine-tuning)

### **Тип: Тестирование безопасности**
- **Основной**: **Promptfoo** (adversarial тесты, 100+ attack vectors, open-source)
- **Альтернатива 1**: **Garak** (NVIDIA, для сканирования уязвимостей LLM)
- **Альтернатива 2**: **LLM Guard** (Content moderation, валидация выхода)
- **Альтернатива 3**: **Rebuff** (специализация на prompt injection)

### **Тип: Генерация синтетических данных**
- **Основной**: **SDV (Synthetic Data Vault)** (табличные данные, статистические паттерны)
- **Альтернатива 1**: **gretel-ai/synthetics** (дифференциальная приватность, криптографический proof)
- **Альтернатива 2**: **Mostly AI** (enterprise, но есть on-premise версия)
- **Альтернатива 3**: **YData Synthetic** (простой, хорош для временных рядов)

### **Тип: Хранилище секретов**
- **Основной**: **HashiCorp Vault** (стандарт, KMS, динамические секреты)
- **Альтернатива 1**: **AKeyless** (SaaS, но есть on-premise)
- **Альтернатива 2**: **Confidant** (Lyft, простой, но без advanced features)
- **Альтернатива 3**: **Infisical** (open-source, современный UI)
