# Contract: Pilot Cutover

## Purpose

Define the minimum proof needed before the repo can consider broader rollout.

## Pilot Preconditions

- Inventory completed
- No unresolved critical blockers for the pilot worktree
- Rollback package prepared
- New operator/review surface defined

## Pilot Success Conditions

- A representative issue lifecycle completes under the new contract
- No silent return to legacy JSONL-first workflow occurs
- Legacy-only surfaces are explicitly blocked or redirected
- Human/agent review remains practical after the lifecycle

## Pilot Failure Conditions

- Any hidden fallback to legacy mode
- Any unresolved ambiguity about the active operator path
- Any loss of reviewability or issue-state consistency
