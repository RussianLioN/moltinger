# Knowledge Base Instructions

This directory is for durable, reusable project knowledge.

## Scope

Use the existing structure:
- `concepts/`
- `patterns/`
- `references/`
- `troubleshooting/`
- `tutorials/`

## Rules

1. Store durable knowledge here, not session chatter.
2. Prefer curated summaries over raw dumps.
3. Do not put secrets, credentials, or environment-sensitive values here.
4. Do not use this directory for temporary investigation notes.
5. Keep entries easy to scan and worth rereading.
6. If knowledge belongs in RCA, reports, or plans instead, put it there instead of here.

## Placement Guidance

- `concepts/` for stable mental models
- `patterns/` for repeatable implementation or operational patterns
- `references/` for compact factual reference material
- `troubleshooting/` for symptom -> diagnosis -> action guides
- `tutorials/` for step-by-step learning material

## Validation

Before adding a file, ask:
- is this durable?
- is this reusable?
- is this the best directory for it?
- does it duplicate an existing doc?

If the answer is mostly no, it likely belongs elsewhere.
